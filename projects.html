Dungeon Quest - Utopia project

Description

Utopia project was a task where our group was meant to created something that would let people experience some aspects of computer science in a fun interactive way. We decided to make a dungeon navigation game operated by writing commands as inputs. The commands resembled the structure of lines of code commonly found in programming languages. Overall, it was meant to simulate a command line operated environment which are frequent in the computer science field. I worked mostly on the input parser, and command identification and execution, but I also helped out with the menus, error handling, structuring the code, and testing.

Reason

This project was part of a Professional skills 1 course at Thomas More. The purpose of this project was to make an interactive method for people to identify the field they would enjoy studying. This project also sparked an interest in game development outside of game engines.

What I learned

Thank to this project, I learned working and structuring a medium size programming project. Since it was a group project communication and teamwork were important. The project background was given to us by a client who was later the judge of it. This thought me how to develop more based on the client's vision than my own vision. From the technical side, I improved my understanding of Pygame which is a Python game making library. Also, since it was a medium sized project, my documentation in a form of code comments improved, so that others would understand code that I have written.

DFS (Depth First Search) Maze generation

Description

This project covers a process of generating a maze using randomized DFS. The process involves treating the maze as a grid of cells and carving out paths between cells in a randomized manner which creates a maze that has a single connected path with no loops. The maze generation uses randomized DFS as the algorithm to create the maze. Depth First Search (DFS) is an algorithm used in computer science for traversing or searching through data structures, for example trees and graphs. The algorithm is placed in a basic Pygame game loop for visualization. Lastly, after the application window is exited, I use cv2 the make and save an image out of the maze. If you want to read more about the project, please visit the GitHub link below.

Reason

Reason for this project was that I wanted to practice programming of some basic algorithm, like the DFS. These basic algorithms are wide used in many different computer science disciplines like artificial intelligence, data science, networking, or bioinformatics.

What I learned

I improved my knowledge on implementing algorithms in code.

MEMENTO - Requirements analysis

Description

The MEMENTO project was a group project in which we create an application analysis for a client. I took the role of a team lead in this group. In this role, I had to mainly made decisions about the work allocation, but I still worked on all sorts of tasks other than just management like working on the Use case diagram or writing descriptions for the document. Our group was well balanced, and communication was not an issue, so most decisions were made collectively. Our group was talking to a potential customer, and we were made to analyse the customers vision for the application by producing a report. Specifically, we were talking to a person with memory issues. They wanted us to create a calendar application meant to help people with those memory issues in day to day life. The focus group of the application were people from 18 to 50 with those issues and people in their circles. For the report, our task was to provide background information, create a Use case diagram with descriptions, a non-functional requirements analysis, provide a priority by functionality using the MoSCoW method, and a high-fidelity prototype made in Axure.

Reason

This project was part of a Requirement analysis course at Thomas More. As described in the description, in this course we talked to a potential customer and made a analyse the customers vision for the application by producing a report. This report can then be used by software developers to create the application.

What I learned

As a team lead of our group, I learned more about managing work group by developing a better communication and managerial skills. The role also improved my understanding of how to better communicate with clients. I also got better at some technical skills like creating a Use case diagram using StarUML and creating an overall analysis of a potential project.

RsPass - Password manager

Description

RsPass is a secure, cross-platform password manager built in Rust, designed for use from a command line. It encrypts your credentials using AES-256-GCM and protects your master password with Argon2 hashing. If you want to read more about the project, please visit the GitHub link below.

Reason

I wanted to practice writing more Rust code which results in an actual usable application.

What I learned

This was the first time I made a command line program in Rust, so I learn had to learn how to handle additional arguments from the command line. Additionally, since the application is storing passwords, it needs to be secure. So, I had to refresh my knowledge on encryption and hashing, and then figure out how to implement it into code.
